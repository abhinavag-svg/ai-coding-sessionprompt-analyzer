# SPEC: AI Coding Prompt Optimizer

## 1. Purpose

AI Coding Prompt Optimizer evaluates Claude coding session efficiency from JSONL logs and produces practical, evidence-based reports that help reduce token waste.

Primary objective:

- Identify and prioritize prompt/process patterns that increase unnecessary cost and correction loops.

For the full V2 scoring model, anti-pattern catalog, detection logic, and design decisions see [`TECHNICAL_SPEC.md`](TECHNICAL_SPEC.md).

## 2. Scope

Included:

- JSONL log parsing and normalization
- Token/cost estimation with configurable pricing profiles
- Deterministic feature extraction and rule evaluation
- V1 and V2 report generation
- Optional local LLM recommendation enrichment (Ollama)

Not included:

- Cloud-hosted inference APIs in core flow
- Automatic model downloads
- Runtime interception/proxy mode

## 3. Inputs

Required:

- A directory containing one or more `.jsonl` session logs

Optional:

- Pricing file (`--pricing-file`)
- Scoring config (`--scoring-config`)
- Recommendation runtime config (`--llm-model`, `--llm-endpoint`, `--llm-timeout-sec`)

## 4. Pipeline

1. Parse JSONL events
2. Normalize usage/cost/event metadata
3. Extract turn and session features
4. Evaluate deterministic rules
5. Compute score outputs
6. Render CLI and Markdown reports
7. Optionally enrich with local LLM recommendations

## 5. Outputs

Report includes:

- Session metrics (turns, tokens, cost, correction ratios)
- Dimension scores and weighted rollups
- Anti-pattern flags with evidence and linked deductions
- Cost confidence and pricing mode attribution
- V2 project rollup and per-session post-mortems

### Anti-Patterns Detected

The tool detects nine named anti-patterns. Each maps to one or more scoring dimensions and carries a concrete remedy.

| Pattern | Severity | What It Catches |
|---|---|---|
| `error_dump` | 🔴 High | Stack traces or HTTP headers pasted in full instead of trimmed to the relevant error |
| `repeated_constraint` | 🔴 High | Same standing rule or constraint stated in 3+ separate turns instead of a system prompt |
| `correction_spiral` | 🔴 High | 3+ consecutive correction turns with no tool use — Claude stuck, session spinning |
| `abandoned_session` | 🔴 High | Session ends on a correction turn with no convergence signal |
| `vague_opener` | 🟡 Medium | Opening prompt under 20 tokens followed by a correction before any tool use |
| `file_thrash` | 🟡 Medium | Same file read more than twice in a session — context loss signal |
| `prompt_duplication` | 🟡 Medium | Prompt body repeated verbatim in the same message — usually a pipeline bug |
| `scope_creep` | 🟡 Medium | Session over 300 turns with tool variety still expanding late |
| `constraint_missing_scaffold` | 🟡 Medium | Specific prompt with file paths still triggers a correction — scaffolding needed |

Optional recommendation section:

- Project-level buckets:
  - `you_did_well`
  - `absolutely_must_do`
  - `nice_to_do`
- Session-level recommendations (only when explicitly enabled)

## 6. Recommendation Behavior

When `--llm-recommendations` is used:

- Project-level recommendations are generated by default.
- Output is constrained to structured buckets.
- Non-actionable caching advice is excluded.

When `--llm-session-recommendations` is used:

- Session-level recommendations are additionally generated.

Failure handling:

- If Ollama/model is unavailable, report generation continues with recommendation status notes.

## 7. CLI Contract

Primary commands:

- `ai-dev analyze`
- `ai-dev analyze-v2`
- `ai-dev cost-range`

Recommendation-related options (V2):

- `--llm-recommendations`
- `--llm-session-recommendations`
- `--llm-model`
- `--llm-endpoint`
- `--llm-timeout-sec`
